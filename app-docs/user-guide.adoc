== Purpose

AbyssL Translator is a Flutter desktop app for translation, spelling correction, rewriting, and document processing with local or remote LLMs.
It supports sufficiently compatible OpenAI Chat Completions and Anthropic Messages endpoints, including cloud providers, gateways, and local servers.
It is designed for fast work with text and files:

* Enter text in `Source` or capture selected text from another app with the global shortcut where the platform adapter supports it
* Read, edit, and copy the result in `Translation`
* Rewrite selected translation text with AI alternatives
* Correct and rewrite longer text in `Correction` mode
* Translate or correct documents through a drop zone, file/folder picker, and batch processing
* Use reasoning options from LM Studio metadata when the local server provides them

== Main Window

The current Translator workspace keeps the instruction, source text, language controls, and translation result in one focused workflow.

The main window uses a navigation rail with these modes:

* `Translator`
* `Correction`
* `Documents`

In `Translator` mode, the window has these areas:

[cols="1,3", options="header"]
|===
| Area | Purpose

| App bar
| App title and `Settings`.

| Language and style row
| Source language, target language, register, complexity, and spelling controls.

| Direct instruction
| Optional instruction for the next translation request.

| Source
| Source text editor.

| Translation
| Target text. The text can be edited, selected, and copied to the clipboard.

=== Source

Use `Source` for the input text.
On app startup it becomes active automatically, so you can type immediately.

Common actions:

* Enter text. If automatic translation is enabled, the app translates after a short debounce.
* Choose the source language: `Detect language` or a fixed source language.
* Clear the text with the clear button; this clears Source, Translation, Synonyms, Alternatives, and status text.

=== Translation

`Translation` shows the current result and remains editable.

Common actions:

* Choose the target language.
* Select text and use `Alternatives`.
* Use the copy button to copy the full translation to the clipboard.

== Translating Text

=== Automatic Translation

When automatic translation is enabled, the app starts a translation after input changes in `Source`.
Input is debounced so a request is not sent for every single character.

=== Manual Translation

Start a manual translation with the `Translate` button.

=== Global Capture Shortcut

The app configures a platform capture adapter with a modifier plus one alphanumeric key.
The default display value is `Ctrl+C`, representing a double-press trigger.
The current macOS and Windows adapters trigger after two matching key presses in a short time window, for example `Ctrl+C+C`.
When text is selected in another app:

. Select text.
. Trigger the configured capture shortcut according to the current platform adapter.
. If capture succeeds, the selected text is copied into `Source`.
. If automatic translation is enabled, AbyssL starts translation immediately.

[NOTE]
====
Global capture is platform-dependent.
On macOS, Accessibility permission and possibly Input Monitoring are required.
On Windows, the native adapter uses a keyboard hook and clipboard copy.
On Linux, this build reports capture status and does not claim a generic Wayland/X11 capture path; use manual copy/paste when the adapter reports unsupported behavior.
====

=== Change the Shortcut

Configure the shortcut in `Settings` with `Capture modifier` and `Key`.
The settings are:

* Modifier: `Control`, `Option`, `Command`, or `Shift`
* Key: one alphanumeric character

The displayed shortcut is `Modifier+Key`.
The exact global-capture behavior depends on the active platform adapter.

== AI Alternatives and Direct Instructions

image::alternatives-workflow.svg[Alternatives workflow, width=900]

=== Alternatives for Selected Text

. Select a word or sentence in `Translation`.
. Click `Alternatives`.
. The app requests the configured number of suggestions.
. The suggestions appear as chips in `Alternatives`.
. Click a suggestion chip to copy it to the clipboard.

=== Alternatives with a Direct Instruction

The Alternatives area includes an `Alternative instruction` input field after synonyms or alternatives are available.
Example:

[source,text]
----
make it more polished and remove filler words
----

Behavior:

* If text is selected in `Translation`, only that selection is rewritten.
* If nothing is selected, the whole current translation is used as the base text.
* The number of suggestions comes from the configured AI alternative count.

=== Copy a Suggestion

Click a visible suggestion chip to copy it to the clipboard.
The current Flutter UI does not automatically replace selected translation text when a suggestion chip is clicked.

== Correction and Rewriting

`Correction` mode is intended for longer text that should be corrected or stylistically rewritten.

image::app-correction.jpg[AbyssL Correction workspace with marked changes and issue cards, width=1100]

=== Correct Text

. Select `Correction` in the navigation rail.
. Paste or type text into the large editor.
. Optionally enter a direct AI instruction.
. Start `Correct`.

The app marks corrected ranges in the input and output text.
Issue cards show the original and corrected text, the model-provided message, and alternatives when the response includes them.

The number of correction alternatives is configured in `Settings`.
The minimum value is 3.

=== Rewrite Text

The rewrite button uses the selected style preset and optional direct AI instruction.
The available style presets are:

* Standard
* Formal
* Concise
* Ultra Short

Example direct instruction:

[source,text]
----
make the text sound like it is from the 17th century
----

=== Copy Corrected Text

The copy button in `Correction` mode writes the current corrected or rewritten text to the clipboard.

== Translating and Correcting Documents

`Documents` processes files or folders.
Depending on the selected options, it corrects, translates, or does both.
When both actions are enabled, correction runs first and translation runs afterward.

image::app-documents.jpg[AbyssL Documents workspace with queued text file and successful PDF result, width=1100]

=== Add Files

You can add documents in these ways:

* Drag files from Finder into the drop zone
* Drop multiple files at once
* Drop a folder; supported files are collected recursively
* Use the file/folder picker button

Unsupported files are shown with a concrete reason.

=== Options

Document mode supports:

* Output directory
* Export format
* Target language
* Optional AI instruction
* `Correct`
* `Translate`
* Filename customization with timestamp, target language, or custom suffix

Filename conflicts are numbered automatically; existing files are not overwritten.

=== Input Formats

Supported inputs in the current code path:

* TXT
* Markdown
* AsciiDoc
* HTML
* RTF
* PDF
* CSV
* TSV
* DOCX, when `/usr/bin/unzip` is available
* XLSX, when `/usr/bin/unzip` is available
* ODT, when LibreOffice `soffice` is found

Pages and Numbers are not treated as safely available input formats.

=== Export Formats

Currently offered export formats:

* PDF
* DOCX
* ODT, when LibreOffice `soffice` is found
* Markdown `.md`
* HTML
* AsciiDoc `.adoc`
* Plain text `.txt`
* RTF

AsciiDoc can optionally be exported as a ZIP with image files when the internal document model contains images.

=== PDF Behavior

PDF files are extracted as text and exported as reflowed documents.
This is not an overlay of the original PDF.
Scanned PDFs without extractable text are reported as unsupported because OCR is not included.
Embedded PDF images are not reliably extracted in the current implementation.

On macOS, AbyssL uses the built-in PDFKit reader and normally needs no
additional installation. If a macOS build reports that PDF extraction is not
available, update or rebuild AbyssL and restart it; an older app bundle may not
include the native reader.

On platforms without native PDF support, AbyssL looks for either
`pdftotext` (from Poppler) or `mutool` (from MuPDF) on the system `PATH`.
For example, macOS users who need the fallback can install Poppler with:

[source,shell]
----
brew install poppler
----

Restart AbyssL after installing a command-line tool so it receives the updated
`PATH`. Neither a text extractor nor an AI model can read text that exists only
as pixels in a scanned PDF; such files require OCR first.

=== Progress

During document processing, the app shows:

* completed file count
* current file
* remaining file count
* elapsed time

== Settings

=== Open Settings

Open Settings with the gear button.
The current Flutter UI shows one scrollable settings dialog with three sections:

* `General` for appearance, languages, text behavior, shortcut, and editor preferences
* `AI Providers` for all provider connections
* `About` for the app version, project website, and GitHub release checks

=== General

`General` controls the desktop workflow without changing provider credentials:

* Theme mode: `System`, `Light`, or `Dark`
* Application language: `System default` follows the macOS language; English, German, French, Spanish, Portuguese, Italian, Dutch, and Croatian can be selected explicitly
* Automatic translation while typing
* Source and target language defaults
* Register, complexity, and spelling defaults
* Alternative counts and editor font size
* Global capture shortcut modifier and key

Changes are committed with `Save`. `Cancel`, Escape, or closing the dialog restores the values that were active before the dialog was opened.

=== AI Providers

`AI Providers` offers three tabs. Select the desired connection type before entering its settings:

[cols="1,3", options="header"]
|===
| Provider | Description

| OpenAI-compatible
| Uses Chat Completions below the configured base URL. Bearer authentication is the default.

| Anthropic-compatible
| Uses Messages below the configured base URL. `x-api-key` and `anthropic-version: 2023-06-01` are the defaults.

| Local OpenAI-compatible
| Uses a local Chat Completions server. The app can also read LM Studio metadata from `/api/v1/models`.
|===

Each connection provides:

* complete base URL, including `/v1` or a gateway path
* authentication mode: Bearer token, `x-api-key`, or none
* API key when required
* freely editable model ID
* request timeout
* model discovery and a minimal connection test when supported

Anthropic-compatible connections additionally provide the Anthropic API version and maximum output-token setting.
The default base URLs are `https://api.openai.com/v1`, `https://api.anthropic.com/v1`, and `http://localhost:11434/v1`.

image::app-anthropic.jpg[AbyssL AI Providers settings with Anthropic-compatible endpoint, model ID, x-api-key authentication, and an empty API key field, width=1100]

API keys are stored with `flutter_secure_storage`.
If secure storage is unavailable, the app reports the error and does not silently save API keys as plaintext.
For production use, prefer the most restricted key that still supports the required requests.

[NOTE]
====
"Compatible" means that the service implements the selected API format sufficiently for the requested operation.
Providers and gateways differ in their support for optional features such as model catalogs, structured output, reasoning fields, and local metadata.
Use `Test connection` for the exact endpoint and model you intend to use.
====

=== Local OpenAI-compatible Server

The local connection adds:

* optional local API key or unauthenticated access
* model detection
* LM Studio reasoning-metadata detection
* per-model reasoning settings

=== Test Connection and Model Discovery

Use `Test connection` to check the configured base URL, authentication, model, and generation endpoint. The test sends a minimal AI request and may be billable when a cloud provider is selected.

`Load models` uses the configured API format's model catalog when available. A missing model-catalog endpoint does not block a manually entered model ID.

For `Local OpenAI-compatible`, `Detect Local` performs these checks:

. Check the configured local server.
. Load available LLMs from LM Studio metadata when the server provides it.
. If no local model name is set, use a loaded model automatically when there is exactly one unambiguous candidate.

Loaded models are marked in the list with a green dot.

[NOTE]
====
The enhanced local model list is based on `/api/v1/models`.
If a local server does not provide this LM Studio metadata path, normal OpenAI-compatible requests can still work, but loaded-state and reasoning metadata can be unavailable.
====

=== Reasoning Options

The app distinguishes between:

* `Reasoning value ON`
* `Reasoning value OFF`

For local LM Studio metadata, `Refresh` can load the available values automatically.
If no reasoning metadata is present, the app treats the local model as a model without reasoning options.

The current request configuration uses `Reasoning value ON` when the `Reasoning` chip is selected and `Reasoning value OFF` when it is not selected.
Provider-specific request handling decides whether an OpenAI-compatible `reasoning_effort` or an Anthropic-compatible effort option is supported and sent. Unsupported optional fields are not guessed.

=== Local Request Timeout

The local request timeout is configured in seconds.
The default value is 600 seconds.
The value `0` disables the app timeout for local provider requests.

=== About and Updates

`About` shows the AbyssL logo, the installed version, the developer name Daniel Mengel, and the project website.
Use `Open website` to visit https://chardonnay.github.io/AbyssL/.

`Check for updates` queries the current GitHub release. When a newer signed macOS release is available, AbyssL offers the native download, verification, installation, and relaunch flow.
If a release does not contain a compatible signed update asset, the app reports that clearly and does not attempt an unsafe installation.

=== General Settings Reference

All General values are retained between launches. This includes theme mode, application-language override, automatic translation, alternative counts, editor font size, capture shortcut, language defaults, and style defaults.

== Window Behavior

On macOS, AbyssL remembers the main-window size and position under the name `AbyssLMainWindow`.
After you resize or move the window, the next launch restores that frame when it is still reachable on an available display.
If the saved frame would be off-screen, AbyssL starts with a centered usable default size instead.

The Settings UI opens as a scrollable dialog.

== Data Storage

Non-secret configuration is stored locally through `shared_preferences`.
API keys are stored through `flutter_secure_storage`.
Stored values include:

* Provider
* OpenAI-compatible connection data
* Anthropic-compatible connection data
* Local OpenAI-compatible connection data
* Local request timeout
* Capture shortcut
* Languages
* Style options
* Reasoning values
* Automatic translation
* Number of AI alternatives
* macOS main-window position and size

The app does not use its own database.

== Architecture

image::architecture.svg[Architecture overview, width=900]

[cols="1,3", options="header"]
|===
| Component | Responsibility

| `AbyssLApp`
| App root, theme mode selection, and `MainShell` setup.

| `MainShell`
| Main window state, navigation between Translator, Correction, and Documents, request cancellation, and status display.

| `SettingsDialog`
| General, AI-provider, and About settings; connection tests; model discovery; local reasoning settings; font size; and capture shortcut settings.

| `AppSettingsStore`
| Persists non-secret settings with `shared_preferences` and API keys with `flutter_secure_storage`.

| `AbyssLApiClient`
| Provider-neutral facade for translation, alternatives, correction, rewriting, model catalogs, and local reasoning metadata.

| Provider adapters
| Map the shared prompts and results to OpenAI-compatible Chat Completions or Anthropic-compatible Messages.

| `PromptBuilder`
| Builds provider prompts for translation, alternatives, correction, and rewriting.

| `DocumentProcessingService`
| Collects, loads, processes, and exports documents.

| `DesktopCaptureService`
| Bridges Flutter to platform capture adapters through the `org.abyssl.translator/capture` method channel.
|===

== Build, Run, and Validate

=== Debug Build

[source,bash]
----
cd apps/abyssl_flutter
flutter pub get
flutter run -d macos
----

Use `flutter run -d windows` or `flutter run -d linux` on matching Windows or Linux hosts.

=== Release Build

[source,bash]
----
bash scripts/build-macos-release.sh
----

The release helper runs `flutter build macos --release` for `apps/abyssl_flutter` and writes `dist/abyssl_flutter-macos-release.zip`.

For published macOS updates, push a stable tag in the form `vMAJOR.MINOR.PATCH`.
The GitHub workflow requires the repository secret `SPARKLE_PRIVATE_KEY`, signs
the update archive with Sparkle's Ed25519 key, generates `appcast.xml`, and
publishes both files to the matching GitHub Release. Users can then open
`Settings > About`, select `Check for updates`, and let the native macOS updater
download, verify, install, and relaunch AbyssL.

Run platform release builds on matching hosts:

[source,bash]
----
flutter build windows --release
flutter build linux --release
----

=== Validate After Changes

[source,bash]
----
cd apps/abyssl_flutter
flutter analyze
flutter test
flutter build macos --debug
----

Validate Windows and Linux builds on their matching hosts with `flutter build windows --debug` and `flutter build linux --debug`.

== Troubleshooting

=== First Character Is Missing in Source

Check:

. Restart the app.
. Type directly without clicking into `Source`.
. The first character must appear in `Source`.

If this does not happen, focus is not set correctly on the source editor.

=== No Translation

Check:

. Is the correct provider selected?
. Is the selected provider format correct?
. Is the complete base URL correct, including `/v1` or the gateway path?
. Does the selected authentication mode match the provider, and is an API key present when required?
. Is a model ID entered, or is exactly one compatible local model available?
. Is the local request timeout high enough? For very slow local models, test with `0`.
. Run `Test connection` in Settings.
. Read the error in the lower status area.

=== Local Models Are Not Listed

Check:

. Is the local server running?
. Are host and port correct?
. Does the server provide `/api/v1/models`?
. For servers without the LM Studio metadata path, this list can stay empty.

=== Reasoning Refresh Shows No Options

Possible causes:

* The local model does not expose reasoning metadata.
* The entered model name does not match the LM Studio metadata.
* The local server does not provide `/api/v1/models`.

=== Global Shortcut Does Not Work

Check:

. Enable Accessibility permission for the app in macOS.
. Select text in the source app.
. Press the configured shortcut, default `Control+C+C`.
. If nothing happens, restart the app and try again.

=== Document PDF Is Empty or Looks Wrong

Check:

. Does the PDF contain extractable text, or is it a scan?
. On macOS, update or rebuild and restart AbyssL to activate the built-in
  PDFKit reader. No Poppler installation should be needed for a current build.
. On another platform, install Poppler (`pdftotext`) or MuPDF (`mutool`) and
  ensure the executable is on the system `PATH`. On macOS, `brew install poppler`
  provides `pdftotext` as a fallback.
. Test export format `Plain text` to isolate text extraction.
. OCR is not implemented for scanned PDFs in the current version.
. PDF export is a reflow export and does not use the original layout as a background.

=== ODT Is Not Offered

ODT requires LibreOffice `soffice`.
The app looks in these locations:

* `/Applications/LibreOffice.app/Contents/MacOS/soffice`
* `/usr/local/bin/soffice`
* `/opt/homebrew/bin/soffice`
