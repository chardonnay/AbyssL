= AbyssL Translator
:toc:
:toclevels: 2

AbyssL Translator is a Flutter desktop app for translation, spelling correction, rewriting, and document processing through OpenAI-compatible Chat Completions and Anthropic-compatible Messages APIs.

The maintained app path is `apps/abyssl_flutter` for macOS, Windows, and Linux.

The app exposes three workspace modes:

* `Translator` for source-to-target translation and alternatives.
* `Correction` for spelling correction, rewrite presets, and marked text changes.
* `Documents` for batch processing files and folders.

image::translation-flow.svg[Translation flow, width=900]

== API Compatibility

AbyssL supports services, gateways, and local servers that implement one of these HTTP API formats sufficiently for text generation:

* OpenAI-compatible Chat Completions through `/chat/completions` below the configured base URL.
* Anthropic-compatible Messages through `/messages` below the configured base URL.
* `/models` below the configured base URL for optional model discovery and connection checks.
* `/api/v1/models` for optional LM Studio metadata on local OpenAI-compatible connections.

The base URL includes the version path, for example `https://api.openai.com/v1`, `https://api.anthropic.com/v1`, or `http://localhost:11434/v1`.
OpenAI-compatible connections default to Bearer authentication. Anthropic-compatible connections default to `x-api-key` plus `anthropic-version: 2023-06-01`. Authentication can also be disabled for local servers.

Compatibility varies between implementations. Native AWS Bedrock, Google Vertex AI, and Azure authentication or request-signing flows require a compatible gateway and are not implied by the generic HTTP adapters.

== Requirements

Flutter desktop app:

* Flutter SDK with desktop support for the target platform.
* A host platform matching the desktop build target:
  ** macOS host for macOS builds.
  ** Windows host for Windows builds.
  ** Linux host for Linux builds.
* Optional: API key for an OpenAI- or Anthropic-compatible cloud provider.
* Optional: local OpenAI-compatible server such as LM Studio.
* Optional for ODT import/export: LibreOffice `soffice`.
* Optional for non-macOS PDF text extraction: `pdftotext` or `mutool` on `PATH`.

== Build and Run

[source,bash]
----
cd apps/abyssl_flutter
flutter pub get
flutter run -d macos
----

Use the matching desktop device on other hosts:

[source,bash]
----
flutter run -d windows
flutter run -d linux
----

Debug builds:

[source,bash]
----
flutter build macos --debug
flutter build windows --debug
flutter build linux --debug
----

Windows and Linux builds must be produced and validated on their matching hosts.

macOS release build helper:

[source,bash]
----
bash scripts/build-macos-release.sh
----

The helper builds `apps/abyssl_flutter` with `flutter build macos --release` and writes `dist/abyssl_flutter-macos-release.zip`.

=== Signed GitHub Updates

The macOS app uses Sparkle 2.9.4 for sandbox-compatible, signed self-updates. The
public Ed25519 key is embedded in the app; the private key remains in the macOS
Keychain under the account `org.abyssl.abysslFlutter` and must never be committed.

The repository secret `SPARKLE_PRIVATE_KEY` must contain the private key exported
by Sparkle's `generate_keys` tool. A `vMAJOR.MINOR.PATCH` tag then makes
`.github/workflows/build-macos.yml` build the universal app, generate a signed
`appcast.xml`, and publish both files as GitHub Release assets. The app reads the
stable feed at:

`https://github.com/chardonnay/AbyssL/releases/latest/download/appcast.xml`

The first Sparkle-enabled release must be installed manually. Following releases
can be checked and installed from `Settings > About`.

== Validate

[source,bash]
----
cd apps/abyssl_flutter
flutter analyze
flutter test
flutter build macos --debug
----

Manual smoke path:

. Start a compatible local server or configure an OpenAI- or Anthropic-compatible cloud endpoint in `Settings`.
. Start the app with `flutter run -d macos`, `flutter run -d windows`, or `flutter run -d linux`.
. Open `Settings`.
. Select `OpenAI-compatible`, `Anthropic-compatible`, or `Local OpenAI-compatible`.
. Enter a complete base URL, choose the authentication mode, and enter a model ID. For local connections, click `Detect Local` to load available models and reasoning metadata when available.
. Run `Test connection`. This sends a minimal provider request and may be billable for cloud providers.
. Translate a multi-paragraph source text and verify the `Translation` field keeps paragraph breaks and blank lines.
. Open `Correction`, run `Correct`, and verify the issue count plus highlighted input/output changes.
. Open `Documents`, add a supported file, choose an output folder, run processing, and verify progress plus the generated output file.

== Settings and Secrets

Non-secret settings are stored through Flutter preferences.
API keys are stored through OS secure storage by `flutter_secure_storage`.
If secure storage is unavailable, AbyssL reports the error and does not silently persist API keys as plaintext.

Settings include:

* Provider: `OpenAI-compatible`, `Anthropic-compatible`, or `Local OpenAI-compatible`.
* Complete base URL including an optional gateway path.
* Authentication: Bearer token, `x-api-key`, or none.
* Freely editable model ID plus optional model discovery.
* Anthropic API version and maximum output tokens.
* Local model detection and manual local model entry.
* Reasoning ON/OFF values with per-model remembered settings.
* Appearance: `System`, `Light`, or `Dark`.
* Editor font size.
* Global capture shortcut modifier and key.

== Translation Workflow

Translator mode supports:

* Source and target language selection.
* Register, complexity, and spelling controls.
* Direct translation instructions.
* Translation with preserved source formatting where possible, including paragraph breaks, blank lines, line breaks, list markers, and item order.
* Synonyms and alternative suggestions.
* Copy result.
* Clear Source and Translation.
* Cancel the active AI request.

The app requests JSON-compatible translation data through the selected provider adapter.
OpenAI-compatible requests use Chat Completions. Anthropic-compatible requests use Messages with the system prompt in the top-level `system` field and combine returned text content blocks in order.
When a provider rejects an optional structured-output or reasoning field with a compatible 400/422 response, AbyssL retries once without that optional field. Authentication, rate-limit, and server errors are never retried automatically to avoid duplicate cost.

== Correction and Rewrite Workflow

Correction mode supports:

* Direct correction or rewrite instruction.
* `Correct` for spelling, grammar, punctuation, and wording issues.
* Marked erroneous input spans.
* Marked corrected output spans.
* Issue count after correction.
* Correction cards with alternatives when returned by the model.
* `Rewrite` with these presets:
  ** `Standard`
  ** `Formal`
  ** `Concise`
  ** `Ultra Short`
* Clear input and output.
* Copy result.
* Cancel the active AI request.

== Local Provider and Reasoning

Local model behavior:

* `Detect Local` loads available models from `/api/v1/models` when available, falling back to `/v1/models`.
* Loaded LM Studio models are marked in the model dropdown when the metadata provides that information.
* If exactly one loaded model is detected, AbyssL can use it automatically.
* The user can still enter a local model name manually.

Reasoning behavior:

* AbyssL can read supported reasoning values from LM Studio metadata when available.
* Reasoning settings are remembered per local model and restored when the same model is selected again.
* For local OpenAI-compatible models, choosing `off` sends `reasoning_effort: "off"`.
* Choosing `none` omits `reasoning_effort`.
* If a local model does not expose reasoning metadata, set the values manually or keep reasoning disabled.

== Document Processing

Document mode processes individual files, multiple files, or folders recursively.
Unsupported files are shown with a reason instead of being silently ignored.

Supported input formats in the Flutter document path:

* TXT, Markdown, AsciiDoc, HTML, RTF
* PDF with extractable text
* CSV, TSV
* DOCX, XLSX
* ODT when LibreOffice `soffice` is available

Unsupported input formats:

* Pages
* Numbers
* DOC
* Scanned PDFs without extractable text

Export formats:

* PDF
* DOCX
* ODT when LibreOffice `soffice` is available
* Markdown `.md`
* HTML
* AsciiDoc `.adoc`
* Plain text `.txt`
* RTF

PDF text extraction uses native macOS support when available.
On other platforms, AbyssL looks for `pdftotext` or `mutool` on `PATH`.

== Global Capture

The capture shortcut uses a modifier plus a key pressed twice, for example `Control+C+C`.

Platform status:

* macOS: native capture adapter, requiring Accessibility permission and possibly Input Monitoring.
* Windows: native capture adapter using a low-level keyboard hook and clipboard copy.
* Linux: status-only adapter in this build. Wayland does not expose a safe generic global text-capture API to this app, and the X11 capture adapter is not enabled. Use manual copy/paste.

Manual capture validation:

. Open AbyssL.
. Confirm the capture shortcut in `Settings`.
. Select text in a native text editor.
. Trigger the configured capture shortcut.
. Verify AbyssL focuses and receives exactly the selected text.
